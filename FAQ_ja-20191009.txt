1. 全般に関すること

MX Linux はローリング・リリースですか？
いいえ。しかし、なるべくユーザーが再インストールをしなくても済むように、移行の詳細が公開されています。Debian ベースが変更されたり、新しい Xfce バージョンがリリースされたりしたとき (どちらも MX-19 で発生しました)、信頼性の高い使用を継続するためには再インストールが必要です。詳細はユーザーズマニュアルの1.6節、またはウィキペディアの記事を参照ください。

MXのリリースサイクルは？
私たちは毎年公式リリースを行ない、その年をリリース番号に使用しています（例えば、MX-18は2018年末にリリースされました）。私たちは姉妹ディストロである antiX と密接に連携し、Xfce の開発も追跡しているので、リリースは通常年末に行われます。

あるバージョンのMXから別のバージョンへアップグレードする一番良い方法は何ですか？
異なるMXバージョンへの移行方法については、MX Community ウェブサイトの移行方法のページを参照してください。

MXのバージョンはどのくらいサポートされますか？
現在、フォーラムを通じて MX 14 から 19.1 までのバージョンをサポートしています。MX 14 は Debian Wheezy をベースにしており、wheezy LTS は 2018 年 5 月 31 日にサポートを終了しました。MX 15/16 もまた数年間サポートされる予定です。現在は、そのバージョンにもできるだけ新しいパッケージを展開しようとしていますが、年月を経過するにつれて実用的でなくなってきています。一般的なサポート情報はユーザーズマニュアルの 1.4 節にあります。

他の多くのディストロが32ビット版を削除しています。MXの場合はどうですか？
MXは、当面の間、32ビット版のリリースを中止する予定はありません。

MX Linux は公式版を他にもリリースする予定がありますか？
いいえ。でもコミュニティによるリスピンが多数あります。詳しくは フォーラムの一覧 参照。

Flatpak や Snaps、Appimage を利用できますか？
スタンドアローンのパッケージは、特定のOSから基本的に独立したものなので、インストールの必要がないという点で異なっています。詳細については、以下のページを参照してください。
• Appimage
• Flatpak
• Snap

MXはソーシャルメディアを使っていますか？
MX にはとても活発なサイトがあります。詳細は、 この一覧 をご覧ください。

MX LinuxにはIRCチャンネルがありますか？
いいえ、私たちはIRCチャンネルをホストしていません。


2. ISO イメージの入手とインストール

なぜインストールが94%で止まってしまうのですか？
あなたは、画面上で求められた情報を入力する必要があるので、インストーラーはそれを待っています。「次へ」を押すと、すべての情報が表示されて先へ進みます。

MX Linuxがインストール済みのUSBは買えますか？
残念ながら、そのサービスを提供している会社が辞めてしまったので、その選択肢はなくなってしまいました。

Windows を搭載したコンピュータに MX Linux をインストールできますか？
マシンに Windows 8 以降が既にインストールされている場合、 (U)EFI や最もよく知られた機能である Secure Boot の存在に対処するために、特別な手順を踏む必要があります。ほとんどのユーザーは、マシンの起動時に BIOS を入力してセキュアブートをオフにするように促されます。通常はインストーラーが処理を行いますが、残念ながら、その後の正確な手順はメーカーによって異なる場合があります。問題がある場合は、ユーザーズマニュアルのセクション 2.4.1 のビデオやドキュメントへのリンクを確認するか、フォーラムでアドバイスを求めてください。

デスクトップにインストール用のアイコンがないので、どうしたらいいですか？
これは時々起こります、特に個人のスナップショットでは、これが時々起こります。F4 を押してターミナルを起動し、次のように入力します。

sudo -E /usr/sbin/minstall


3. 各種設定とカスタマイズ

サイドにあるパネルが嫌です！ 希望の場所へ移動させるにはどうしたらいいですか？
MX Tweakというツールを使えば、お好みの位置へ非常に簡単に移動できます。

マルチメディアに必要なコーデックをインストールするにはどうすればいいですか？
MX コーデック・インストーラーを使ってください。

XFCE は好みではありません。他のデスクトップ環境は使えますか？
MX パッケージ・インストーラーには、「ウィンドウマネージャー」というカテゴリの下に、さまざまなデスクトップが用意されているので、簡単にインストールすることができます。

XFCE環境で他のデスクトップ用のアプリを使えますか？
既定のリポジトリにある XFCE 以外のウィンドウマネージャーのアプリケーションは、インストールすることができます。多くの場合、アプリケーションの依存関係にある他のアプリケーションやライブラリがインストールされます。場合によっては、インストールされたアプリケーションの全機能が利用できないことがあるので、その機能を有効にする方法を調査する必要があります。また、特に Xfce 4.14 へのアップグレード以降は、アプリケーションの外観が XFCE の既定のテーマと一致しないことがあります。

ウィンドウの縁取りが細すぎて、簡単にウィンドウのサイズ変更ができません。何か方法はありませんか？
Alt + F8 キーを使用して、右下のウィンドウの角をつかみ、希望のサイズにドラッグします。または、より大きなボーダーを持つテーマ（設定 > 外観）を選択します。

パネルにワンクリックでログアウトやシャットダウン、再起動できるボタンを追加するにはどうすればいいですか?
パネルの上部にある（緑色の）アクションボタンのアイコンを右クリックして、その中から「プロパティ」を選択し、確認したいもの以外の全てのチェックを外します。確認のチェックを外したままにしておくと、さらに速くなります。


4. セキュリティについて

スペクターやメルトダウンの話をよく聞きますが MXは安全ですか？ どうやって自分のシステムを確認するのですか?
root ターミナルで spectre-meltdown-checker を実行することで、使用中のカーネルの状態を確認することができます。Synaptic または MX パッケージインストーラを使って、既定の Debian リポジトリからパッケージをインストールする必要があります。パッチが適用されたカーネルは、そのステータスの説明と共に、MX パッケージインストーラ > カーネルカテゴリの中の「人気のあるパッケージ」から入手できます。

Ubuntu PPA は使えますか？
Ubuntu は Debian ではないので、PPA をインストールしたらおそらく問題が発生します。詳しくは こちらの記事 をご覧ください。解決方法：コミュニティリポジトリ (CR) の下にあるあなたの MX バージョンのリクエストフォーラムに行き、私たちのパッケージ開発者に、その PPA を移植したり、私たち自身のリポジトリ用に更新したりできるかどうかを尋ねてみてください。

MXテスト版や Debian実験版、マルチメディアなどのリポジトリを有効にしておくべきですか？
これらのリポジトリを恒常的に有効にしておくと、システムが不安定になり、再インストールが必要になる可能性があります。解決策： MXツール > パッケージインストーラ を使用して、選択したリポジトリよりパッケージを個別にインストールしてください。

なぜ Flashがインストールされているのですか？
MX Linux は基本的にユーザー志向なので、システムが out-of-the-box で最大限動作可能なことを保証するために、ある程度のノンフリーなソフトウェアが含まれています。Flash は、パッケージインストーラを使って簡単にアンインストールできます。


5. システムについて

MX と systemd の対応はどうなっていますか？どうして systemd のパッケージもインストールされているのですか？
Systemd はいくつかの重要なアプリケーションを実行できるようにする目的で含まれていますが、既定では有効になっていません。詳細については、ユーザーマニュアルのセクション1.7と Wiki の記事 を参照してください。

ネットワークマネージャーを使ってランダムログインの問題を解決するには？
これは既知のバグが原因です。詳しくは Wiki のこの記事 を見てください。

systemd を既定として起動できる方法はありますか？
はい。開いている（GRUBとして知られている）画面の "高度なオプション" の行をクリックしてsystemd を起動し、MX Boot オプションを起動し、「直前に選択した起動の保存を有効化」のボックスにチェックを入れます。

どのパッケージマネージャーを使ったら良いのですか？
グラフィカルなパッケージマネージャーは２つあります：
• MX パッケージマネージャー (MXPI) は、人気あるアプリや、Debian安定版、MXテスト版、Debianバックポート、Flatpak リポジトリのアプリをワンクリックでインストール／削除ができます。
• Synaptic パッケージマネージャーは、Debian パッケージを包括的に管理できるツールです。
こうしたこと及びコマンドラインのユーティリティの詳細については、ユーザーマニュアルのセクション5を参照してください。

アップデートがあったことをどうやって知ることができますか？
通知領域に浮かび上がった小さなボックスが緑色に変わります。それをクリックすると、新しいパッケージが表示されます。
